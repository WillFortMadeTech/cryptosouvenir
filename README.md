## Local Development

1. Copy the example environment file and fill in your values:
   ```bash
   cp .env.example .env
   ```
2. Start the app via Docker Compose (reads `.env` automatically):
   ```bash
   docker compose up
   ```
   The app will be available at [http://localhost:3000](http://localhost:3000).

> All variables use the `NEXT_PUBLIC_` prefix so they are accessible to both the Next.js server and the browser bundle. The app ships with sensible defaults and will run without a `.env` file, but defining one lets you tune grid resolution, cube appearance, and rendering flags without touching source code.
